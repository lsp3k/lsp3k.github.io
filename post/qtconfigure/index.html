config_help.txt

安装目录

-prefix <dir/>目标设备上的部署目录(deployment directory)
-extprefix <dir>主机上的安装目录 (installation directory)
-hostprefix [dir]主机上构建工具的安装目录(installation directory for build tools)如果没有设定，则使用当前目录
-external-hostbindir此机器的Qt工具编译路径当-platform与当前系统不匹配时使用

Fine tuning of installation directory layout

注意所有目录除了 -sysconfdir，其他都应位于 -prefix/-hostprefix 下

-bindir <dir>Executables [PREFIX/bin]
-headerdir <dir>Header files [PREFIX/include]
-libdir <dir>Libraries [PREFIX/lib]
-archdatadir <dir>Arch-dependent data [PREFIX]
-plugindir <dir>Plugins [ARCHDATADIR/plugins]
-libexecdir <dir>Helper programs [ARCHDATADIR/bin on Windows，ARCHDATADIR/libexec otherwise]
-importdir <dir>QML1 imports [ARCHDATADIR/imports]
-qmldir <dir>QML2 imports [ARCHDATADIR/qml]
-datadir <dir>Arch-independent data [PREFIX]
-docdir <dir>Documentation [DATADIR/doc]
-translationdir <dir>Translations [DATADIR/translations]
-sysconfdir <dir>Settings used by Qt programs [PREFIX/etc/xdg]
-examplesdir <dir>Examples [PREFIX/examples]
-testsdir <dir>Tests [PREFIX/tests]
-hostbindir <dir>Host executables [HOSTPREFIX/bin]
-hostlibdir <dir>Host libraries [HOSTPREFIX/lib]
-hostdatadir <dir>Data used by qmake [HOSTPREFIX]

Configure meta

-help, -h显示帮助
-verbose, -v配置过程打印详细信息
-continue忽略错误继续配置
-redo使用先前的参数重新配置
-recheck [test,…]丢弃所有否定配置测试结果缓存在安装缺少的依赖项后使用它
-recheck-all丢弃所有配置测试结果缓存
-feature-<feature>Enable <feature>
-no-feature-<feature>Disable <feature> [none]
-list-features列出可用功能某些功能也有专用命令行选项
-list-libraries列出可能的外部依赖项

编译选项

-opensource构建 Qt 的开源版本
-commercial构建 Qt 商业版
-confirm-license自动确认许可
-release不包含调试信息
-debug包含调试信息
-debug-and-release编译两个版本
-optimize-debug在调试版本中启用优化 [auto](不支持msvc或者clang工具链)
-optimize-size优化 release 版本的大小而不是速度[no]
-optimized-tools构建优化主机工具即使在 debug 版本
-force-debug-info为 release 版本创建符号文件[no]
-separate-debug-info调试信息拆分为单独文件 [no]
-gdb-index对调试信息进行索引来加速 gdb [no][auto] 如果-developer-build 编译调试信息
-strip剥离 release 二进制文件不需要的符号
-gc-binaries将每个函数或数据项放入其自己的区块，对未使用的区块开启链接器垃圾收集[auto] 静态编译,[no]其他
-force-asserts即使是 release 版本也开启 Q_ASSERT [no]
-developer-build编译和链接 Qt 用于开发 Qt 本身 [no]用于自动测试、额外检查等导出
-shared构建 Qt 共享库 [yes][no] UIKit
-static构建静态 Qt 库 [no][yes] UIKit
-framework构建Qt框架 bundles[yes] (Apple only)
-platform <target>选择主机 mkspec [detected]
-xplatform <target>交叉编译时选择目标 mkspec [PLATFORM]
-device <name>交叉编译的设备名<name>
-device-option <key=value>添加选项到设备 mkspec
-appstore-compliant禁用平台 appstore 中不允许的代码 [auto]
-qtnamespace <name>封装所有的 Qt 库代码在’namespace <name> {…}‘中
-qtlibinfix <infix>重命名所有 libQt5*.so 为 libQt5*<infix>.so
-qtlibinfix-plugins根据 -qtlibinfix 重命名 Qt 插件 [no]
-testcocoon使用 TestCocoon 代码覆盖工具进行检测 [no]
-gcov使用 GCov 代码覆盖工具进行检测 [no]
-trace [backend]启用跟踪点检测
-sanitize {address| thread| memory| fuzzer-no-link | undefined}使用指定的编译器清理程序进行检测
-coverage {trace-pc-guard|source-based}对sanitizers指定’trace-pc-guard’覆盖检测或者生成检测代码去收集计数并启用代码覆盖率分析
-c++std <edition>选择C++标准<edition> [c++2a/c++17/c++14/c++11]不支持MSVC 2015
-sse2使用 SSE2 指令[auto]
-sse3/-ssse3/-sse4.1/-sse4.2/-avx/-avx2/-avx512启用使用特点的 x86 指令[auto]启用的仍然受运行时检测的影响
-mips_dsp/-mips_dspr2使用 MIPS DSP/rev2 指令[自动]
-qreal <type>typedef qreal 指定的类型 [auto]这会影响二进制兼容性(binary compatibility)
-R <string>向 Qt 库添加显式运行时库路径支持路径相对于LIBDIR
-rpath使用库安装路径作为运行时库路径链接Qt库和执行文件 [auto]如同-R LIBDIR，在Apple平台上，禁用此功能意味着使用绝对安装名（依赖于LIBDIR）进行动态库和框架
-reduce-exports减少导出符号的数量 [auto]
-reduce-relocations减少重定位量 [auto] (Unix only)
-plugin-manifests将清单嵌入插件 [no] (Windows only)
-static-runtime使用 -static，使用静态运行时 [no] (Windows only)
-pch使用预编译头文件 [auto]
-ltcg使用链接时间代码生成 [no]
-linker [bfd,gold,lld]强制使用 GNU ld、GNU gold 或 LLVM/LLD 链接器而不是默认链接器 (GCC only)
-incredibuild-xge使用 IncrediBuild XGE [no] (Windows only)
-ccache使用 ccache 编译器缓存 [no] (Unix only)
-make-tool <tool>使用 <tool> 构建 qmake [nmake] (Windows only)
-mp使用多个处理器进行编译 (MSVC only)
-warnings-are-errors将警告视为错误 [no; yes，if -developer-build]
-silent减少构建输出，以便更容易看到警告和错误

编译环境

-sysroot <dir>将 <dir> 设置为目标 sysroot
-gcc-sysroot使用 -sysroot，将 –sysroot 传递给编译器 [yes]
-pkg-config使用 pkg-config [auto] (Unix only)
-D <string>通过额外的预处理器定义
-I <string>传递额外的包含路径
-L <string>传递额外的库路径
-F <string>传递额外的框架路径 (Apple only)
-sdk <sdk>使用 Apple 提供的 SDK <sdk> 构建 Qt
-android-sdk path设置 Android SDK 根路径 [$ANDROID_SDK_ROOT]
-android-ndk path设置 Android SDK 根路径 [$ANDROID_SDK_ROOT]
-android-ndk-platform设置 Android 平台
-android-ndk-host设置 Android NDK 主机（linux-x86、linux-x86_64 等）[$ANDROID_NDK_HOST]
-android-abis逗号分隔的 Android abis，默认为：armeabi-v7a,arm64-v8a,x86,x86_64
-android-style-assets设备运行时自动解压 style assets [yes]

组件选择

-skip <repo>从构建中排除整个<repo>
-make <part>将<part>添加到要构建的部件列表指定此选项将清除默认列表
-nomake <part>从要构建的部件列表中排除 <part>
-compile-examples未设置时，仅安装示例的源码[no on WebAssembly, otherwise yes]
-gui构建 Qt GUI 模块和依赖 [yes]
-widgets构建 Qt Widgets 模块和依赖项 [yes]
-no-dbus不要构建 Qt D-Bus 模块[default on Android and Windows]
-dbus-linked构建 Qt D-Bus 并链接到 libdbus-1 [auto]
-dbus-runtime构建 Qt D-Bus 并动态加载 libdbus-1 [no]
-accessibility启用辅助功能支持 [yes]不建议禁用辅助功能

Core 选项

-doubleconversion选择使用的双转换库[system/qt/no]
-glib启用 Glib 支持[no; auto on Unix]
-eventfd启用 eventfd 支持
-inotify启用 inotify 支持
-iconv启用 iconv(3) 支持 [posix/sun/gnu/no] (Unix only)
-icu启用 ICU 支持 [自动]
-pcre选择使用 libpcre2 [system/qt/no]
-pps启用 PPS 支持 [auto] (QNX only)
-zlib选择使用 zlib [system/qt]

Logging backends

-journald启用日志支持 [no](Unix only)
-syslog启用syslog支持 [no](Unix only)
-slog2启用slog2支持 [no](QNX only)

Network 选项

-ssl启用任一 SSL 支持方法 [auto]
-no-openssl不使用OpenSSL[default on Apple and WinRT]
-openssl-linked使用 OpenSSL 并链接到 libssl [no]
-openssl-runtime使用 OpenSSL 并动态加载 libssl [auto]
-schannel使用安全的信道 [no] (Windows only)
-securetransport使用 SecureTransport [auto] (Apple only)
-sctp启用 SCTP 支持 [no]
-libproxy启用 libproxy [no]
-system-proxies默认使用系统网络代理 [yes]

Gui, printing, widget 选项

-cups启用 CUPS 支持 [auto] (Unix only)
-fontconfig启用 Fontconfig 支持 [auto] (Unix only)
-freetype选择使用 FreeType [system/qt/no]
-harfbuzz选择使用 HarfBuzz-NG [system/qt/no](Not auto-detected on Apple and Windows)
-gtk启用 GTK 平台主题支持 [auto]
-lgmon启用 lgmon 支持 [auto] (QNX only)
-no-opengl禁用 OpenGL 支持
-opengl <api>启用 OpenGL 支持支持的接口： es2 (default on Windows), desktop (default on Unix),dynamic (Windows only)
-opengles3启用 OpenGL ES 3.x 支持而不是 ES 2.x [auto]
-egl启用 EGL 支持 [auto]
-angle使用 bundled ANGLE 来支持 OpenGL ES 2.0 [auto]
-combined-angle-lib将 LibEGL 和 LibGLESv2 合并到 LibANGLE (Windows only)
-qpa <name>选择默认的 QPA 后端(e.g., xcb, cocoa, windows)
-xcb-xlib启用 Xcb-Xlib 支持 [auto]

Platform backends

-direct2d启用 Direct2D 支持 [auto] (Windows only)
-directfb启用 DirectFB 支持 [no] (Unix only)
-eglfs启用 EGLFS 支持 [auto; no on Android and Windows]
-gbm启用 backends for GBM 支持 [auto] (Linux only)
-kms启用 backends for KMS 支持 [auto] (Linux only)
-linuxfb启用 Linux Framebuffer 支持 [auto] (Linux only)
-xcb启用 X11 支持 [auto] (Linux only)

Input backends

-libudev启用 udev 支持 [auto]
-evdev启用 evdev 支持 [auto]
-imf启用 IMF 支持 [auto]
-libinput启用 libinput 支持 [auto]
-mtdev启用 mtdev 支持 [auto]
-tslib启用 tslib 支持 [auto]
-bundled-xcb-xinput使用 bundled XInput2 支持 [auto]
-xkbcommon启动键映射支持 [auto]

图片格式

-gif启用对 GIF 的读取支持 [auto]
-ico启用支持 ICO [yes]
-libpng选择使用 libpng [system/qt/no]
-libjpeg选择使用 libjpeg [system/qt/no]

数据库选项

-sql-<driver>启用SQL <driver> 插件支持的驱动程序db2 ibase mysql oci odbc psql sqlite2 sqlite tds [all auto]
-sqlite选择使用 sqlite3 [system/qt]

常用编译命令

  • 动态链接编译 debug 和 release （安路径为D:\qt\library\qt-everywhere-src-5.15.0-shared-x86）

    1

    configure.bat -prefix "D:\qt\library\qt-everywhere-src-5.15.0-shared-x86" -confirm-license -opensource -debug-and-release -platform win32-msvc -shared -nomake examples -nomake tests -opengl desktop -mp

  • 静态链接编译 debug 和 release （安装路径为D:\qt\library\qt-everywhere-src-5.15.0-static-x86）

    1

    configure.bat -prefix "D:\qt\library\qt-everywhere-src-5.15.0-static-x86" -confirm-license -opensource -debug-and-release -platform win32-msvc -static -static-runtime -nomake examples -nomake tests -nomake tools -opengl desktop -mp

MSVC VS2019 编译 qt-everywhere-src-5.15.0

  1. 下载解压 qt-everywhere-src-5.15.0.zip
  2. 打开 x86 Native Tools Command Prompt for VS 2019 （编译64位打开x64 Native Tools Command Prompt for VS 2019）
  3. cd 到 qt-everywhere-src-5.15.0 目录下执行上面编译命令
  4. nmake
  5. nmake install
